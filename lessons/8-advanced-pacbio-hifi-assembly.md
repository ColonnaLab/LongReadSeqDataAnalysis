## Advanced 2: PacBio HiFi genome assembly

This advanced session is the PacBio HiFi counterpart to the ONT assembly hackathon in lesson 7. The goal is to assemble high-accuracy long reads, evaluate the assembly, and compare it to a complete reference genome.

The practical challenge is in [the PacBio HiFi assembly hackathon brief](8-advanced-pacbio-hifi-assembly-hackathon.md).

Dataset:

- organism: *Escherichia coli* K-12
- SRA accession: `SRR10971019`
- data type: PacBio HiFi / CCS reads
- expected genome: approximately 4.64 Mb
- expected genome structure: one circular chromosome
- reference accession: `NC_000913.3`
- reference assembly: `GCF_000005845.2`
- full download size: approximately 3 GB
- full coverage: approximately 290x
- server data folder: `/data/user1/data-pacbio`

### **1. Why PacBio HiFi Assembly Is Different**

PacBio HiFi reads are long reads with high per-read accuracy. They are generated by circular consensus sequencing, where the same DNA molecule is read multiple times and converted into a high-quality consensus read.

Compared with raw ONT reads, HiFi reads usually have:

- higher base accuracy
- more reliable small-variant evidence
- shorter maximum read lengths than ultra-long ONT reads
- strong performance for bacterial genome assembly

Because HiFi reads are already accurate, assemblers can use algorithms and parameters optimized for high-accuracy long reads.

### **2. Assembly Goal**

For this dataset, the expected biological result is simple:

- one circular *E. coli* K-12 chromosome
- approximate size: 4.64 Mb
- no plasmid expected for the reference genome

The assembly may still produce more than one contig if:

- coverage is uneven
- repeats are unresolved
- reads were filtered or subsampled too strongly
- assembler parameters are not appropriate
- contamination or extra sequences are present

### **3. HiFi Assembly Tools**

Common PacBio HiFi assembly tools include:

- [`hifiasm`](https://github.com/chhylp123/hifiasm)
  - widely used for PacBio HiFi reads
  - produces assembly graph output in GFA format
  - often used for high-accuracy long-read assemblies
- [`Flye`](https://github.com/mikolmogorov/Flye)
  - supports PacBio HiFi reads with `--pacbio-hifi`
  - useful for comparison with the ONT assembly session
- [`HiCanu`](https://github.com/marbl/canu)
  - HiFi mode of the Canu assembler
  - useful for high-accuracy long reads

In this hackathon, `hifiasm` is the recommended first assembler, with `Flye --pacbio-hifi` as a useful comparison.

```diff
+ HiFi reads are not ONT reads
+ Do not use ONT-specific assembler options such as --nano-raw for this dataset
```

### **4. Assembly Evaluation**

The main evaluation question is whether the assembly recovers one complete *E. coli* chromosome.

Useful questions:

- How many contigs were produced?
- Is the total assembly length close to 4.64 Mb?
- Is the longest contig close to chromosome length?
- Is the assembly N50 close to total genome size?
- Does the assembly align across the full `NC_000913.3` reference?
- Are there apparent duplications, missing regions, or rearrangements?

Useful tools:

- `seqkit stats`: FASTA/FASTQ statistics
- `QUAST`: assembly quality and reference comparison
- `minimap2`: assembly-to-reference alignment
- `samtools`: BAM sorting, indexing, and coverage
- `IGV`: visual inspection
- `Bandage`: graph visualization for GFA assembly graphs

### **5. Reference Comparison**

The reference for comparison is:

- accession: `NC_000913.3`
- assembly: `GCF_000005845.2`

Reference comparison can answer:

- Was the chromosome recovered in one contig?
- Are there large missing regions?
- Are there duplicated regions?
- Does the assembly have structural differences relative to the reference?

Even with a high-quality reference, remember that assembly-reference differences can be caused by real biological differences, assembly errors, or reference/strain differences.

### **6. Optional Subsampling**

The full dataset is approximately 290x coverage. This is useful for benchmarking, but it may be more than needed for a classroom assembly.

Advanced groups can compare:

- full data assembly
- subsampled assembly
- different assemblers on the same input

Subsampling is optional and should be documented clearly if used.

### **8. Key Points**

- PacBio HiFi reads are high-accuracy long reads
- HiFi assembly should use HiFi-aware assembler options
- For *E. coli* K-12, the expected assembly is approximately one 4.64 Mb chromosome
- Full coverage is high, so subsampling can be a useful advanced comparison
- Assembly evaluation requires both statistics and biological interpretation
